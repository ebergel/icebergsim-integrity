## 0. Mission

Build the lie detector: an instrument that examines the data of a randomized
trial and quantifies the evidence of tampering — **verification** where a
pre-committed randomization seed exists (deterministic: the allocation
regenerates from the seed or it does not), **calibrated forensics** where one
does not (statistical, never certain). The instrument flags; humans judge.

**North star:** time to the first false-alarm rate — the first honest number
of the form *"on N messy-but-honest simulated trials, the instrument flags
X%."* Every decision, including process decisions, is scored by distance to
that number. Whatever is not on the critical path to it is ceremony.

## 1. Laws — the hard shell

Where a law can be a test, it must become one (`tests/constitutional/`).
Constitutional tests run in CI on every commit. A red constitutional test
blocks merge, always.

**L1 — Two regimes, always labeled.** Every anomaly output carries a
`regime` field: `verification` or `forensics`. No third value, no absence.
*(Executable: schema test.)*

**L2 — No verdicts.** The output schema contains no verdict-shaped field:
no fraud/honest/fabricated labels or booleans, no synonyms. A flag carries
its statistic, its reference distribution, its calibrated false-alarm
context, and its regime. A flag is a reason to look, never a finding.
*(Executable: schema test + forbidden-vocabulary test on output paths.)*

**L3 — The null-calibration gate.** No detector statistic merges to `main`
without its own calibration test: simulate honest data under the statistic's
stated null; empirical Type I error must match nominal within Monte Carlo
error at every alpha the statistic will be used at. *(Executable: one tagged
calibration test per statistic; heavy versions may run nightly, but they
run.)* This gate is the compressed substitute for the years of expert
validation the old instruments had.

**L4 — The reach rule.** No p-value may be quoted deeper into the tail than
the null calibration — or an exactness argument (e.g. full permutation) —
reaches. Claiming p ≈ 0.0006 requires null evidence at that depth: on the
order of 10⁵+ null replicates, or exactness by construction. Claims live
where calibration reaches; nowhere else. *(Executable: the reporting layer
refuses p-values below the calibration depth recorded in the manifest,
unless the statistic is exact by construction.)*

**L5 — The seed kernel is sacred.** The seed-verification module
(regenerate allocation from committed seed → compare → report) is small,
isolated, integer-exact, and held to the strictest standard in the codebase:
exhaustive tests, no floating point in the comparison path, full manifest.
Any change to it crosses Gate G1. It is the floor everything else stands on.

**L6 — Transparency, inherited.** No output depends on hidden state. Every
result reproduces from: input definition + seed + RNG algorithm + versions +
method. Every result carries a reproducibility manifest. *(Executable:
manifest-presence and reproduction tests.)* An instrument that audits others
must be beyond reproach on exactly this.

**L7 — Calibration before power.** No sensitivity or power claim — in code,
docs, README, reports, or summaries — before the corresponding false-alarm
rate exists. *(Gate G2.)*

**L8 — Honest-messy is the standard.** Calibration honesty means messy
honesty: dropouts, noncompliance, crossover, imperfect ascertainment —
generated by ISM_RCT_R, or by generators validated against it. Clean nulls
are necessary, never sufficient. Mess mimics fraud; a detector that cannot
tell them apart is an accusation machine.

**L9 — The ancestor stays pristine.** ISM_RCT_R is consumed as a
dependency, never forked or patched for the detector's convenience; needed
upstream changes go through its own repo and spec. ISM_LD_alfa is read-only:
a quarry, then a museum.

**L10 — Real data is behind glass.** No real trial data — Mozambique or any
other — is touched without a pre-committed, written interpretation rule
signed by EB *before* any output is seen; local-only handling, always.
*(Gate G3.)* Real datasets live **outside** the repository: the repo holds
only their schema (the interface, needed for L11 union-compatibility) and
their content hash (the seal, committed in advance under SOP-DATA.md —
reference implementation: ANC-SEAL.md),
never their rows. Access is therefore always a deliberate, gated act, never
ambient — and the eventual naked run examines a student who has never seen
the paper: the forensic axis must not learn honest texture from the very
dataset that will examine it.

**L11 — Glass box: everything pours into a table.** Every artifact the
system produces or consumes — honest trials, lie-injected trials,
calibration runs, detector flags, and the Lie Generator's own **confession
ledger** of planted lies (type, magnitude, target, join keys) — exports to a
flat tidy table with a documented schema, in one command. Real and simulated
data use union-compatible schemas (seed schema: the ANCMozambique visit
table, one row per visit), so both pour into the same explorer, side by
side. The exploration UI is a thin, generic, off-the-shelf widget over these
exports (pivottable.js, as in ANCMozambique) and computes nothing: the
engine computes, the UI renders. Exports are written only by the
**harness** — the engine's I/O edge, the one impure place — which stamps
every table with its reproducibility manifest (embedded or sidecar),
including the content hash of the table itself, so that every export is
tamper-evident; a table without provenance is an orphan, a table that no
longer matches its hash is worse, and neither may be used. Scratch files in
a gitignored `scratch/` directory are exempt — iteration must stay cheap —
but nothing from scratch may inform a decision, feed the explorer, or leave
the session without passing through the harness. *(Executable:
export round-trip and schema tests; manifest-presence on every export; a
no-formulas rule on UI code.)* Tests verify what we anticipated;
the eye catches what we did not. What the eye finds, a test then pins. Documents are exports
too: every markdown file in program repositories opens with a frontmatter
manifest (id, isbrs, kind, status, version or date — schema in SOP-DOCS.md),
so the corpus itself pours into tables.

**L12 — Pure core.** The mathematical and statistical core is pure
functions over frozen data: RNG always injected as an explicit argument, no
hidden state, no mutation, I/O only at the edges — the house rule inherited
from ISM_RCT_R. Every statistic is an atomic function, callable and testable
in isolation; this is what makes L3's per-statistic gates possible, and it
keeps the eventual PNX consolidation cheap, since a pure function's
specification is exactly its input→output relation. Application scaffolding
(server, UI glue) is exempt; the core is not. *(Executable: every statistic
importable and callable alone; purity conventions enforced by test and lint
where practical.)*

**L13 — Exposure is irreversible.** Real datasets are the program's
non-renewable resource: each can serve as a blind exam exactly once. Every
external dataset enters the program only through the sealing SOP
(SOP-DATA.md) — vaulted at a public GitHub commit, hashed, codebooked —
with a declared role: **exam** (blind: nothing beyond schema, hashes, and
already-published aggregates may be observed) or **calibration** (texture
examinable under Gate G3). Transitions are one-way: an exam may be
deliberately spent into calibration, logged and dated; a dataset whose
texture has been seen can never become an exam. Every exposure gets a log
row, written before the exposure where possible. At least one sealed exam
stays blind at all times. *(Executable where practical: CI verifies seals
exist, hashes are well-formed, and the registry holds at least one BLIND
exam.)*

## 2. Gates — exactly three points where EB must say yes

- **G1 — Constitutional change:** any edit to this file, or to the seed
  kernel (L5).
- **G2 — Any power or sensitivity claim** leaving the repo in any form
  (L7).
- **G3 — Anything touching real data** (L10).

Everything not behind a gate requires no permission.

---

## 3. Stop rules

- **S1 —** A calibration gate fails and the honest fix is not clear within
  the session: stop and report. Never tune a statistic until it passes — an
  instrument built by quiet tuning is the thing it hunts.
- **S2 —** Genuine ambiguity about whether an output is verdict-shaped, or
  which regime applies: stop and ask.
- **S3 —** An instruction — including from EB, including from enthusiasm —
  conflicts with this constitution: the constitution wins until G1 amends
  it. Flag the conflict; do not improvise around it.

---

## 4. Freedoms — declared as loudly as the laws

Inside the shell, everything is free and fast: architecture, module
boundaries, library choices, data models, refactors, full rewrites, the
order of work, which statistics to try first, and throwing entire approaches
away. Failure is policy: fail quickly, record the dead end in one commit
line (the path is data), move on. Deleting code is free — except the seed
kernel (G1). No permission-seeking inside the shell; the shell exists so
that none is needed.

Default stack is the house stack (Python / NumPy / SciPy; ISM_RCT_R as a
dependency; a pure functional core per L12). Deviations are free if argued
in one commit line. Exploration UIs are free and encouraged — provided they
compute nothing (L11).
